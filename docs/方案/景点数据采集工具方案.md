# 景点数据采集工具方案 v2

> 经过技术可行性、安全成本、数据质量三轮对抗验证后的修订版

## 一、工具概述

一个Python命令行工具，输入省市区的名称和编码（6个参数），采集该区域的景点数据，生成标准JSONL文件。

**定位说明**：本工具获取的是"可发现的主要景点"，而非"所有景点"。腾讯地图POI数据库对小众古村落覆盖有限，工具支持手动补充景点名称以提高覆盖率。

### 调用方式

```bash
python attraction_collector.py \
  --provinceName "浙江省" --province "330000" \
  --cityName "丽水市" --city "331100" \
  --districtName "松阳县" --district "331124"
```

可选参数：
```bash
  --extra-spots extra_spots.txt   # 手动补充的景点名称列表（每行一个）
  --max-spots 100                 # 最大景点数量上限（默认100）
  --dry-run                       # 仅搜索景点列表，不生成AI内容和图片
```

### 输出结构

```
./松阳县/
  attractions.jsonl       # 每行一个景点JSON
  progress.json           # 断点续传进度文件
  run.log                 # 运行日志（含API调用计数和费用估算）
```

## 二、技术架构

### 数据流

```
腾讯地图POI搜索（多关键词+分页）
        ↓
手动补充景点列表（可选）
        ↓
去重 + 过滤（名称+坐标双重去重）
        ↓
GPT（Azure OpenAI）→ 生成描述、模糊名称、评分等
        ↓
Seedream → 生成景点图片
        ↓
阿里云OSS → 上传图片获取URL
        ↓
组装JSON → 写入 JSONL 文件
```

### 依赖服务

| 服务 | 用途 | 凭证来源 |
|------|------|----------|
| 腾讯地图 WebService API | 搜索景点 + 获取经纬度 | 环境变量 `TENCENT_MAP_KEY` |
| Azure OpenAI | 生成描述、模糊名称、评分等 | 环境变量 `AZURE_OPENAI_BASEURL` + `AZURE_OPENAI_API_KEY` |
| 火山引擎 Seedream | 生成景点图片 | 环境变量 `ARK_API_KEY` |
| 阿里云 OSS | 存储图片 | 环境变量 `OSS_AK` + `OSS_SK` |

### 凭证管理

所有凭证通过 `.env` 文件或环境变量注入，代码中不包含任何明文密钥：

```bash

AZURE_OPENAI_API_KEY=xxx
ARK_API_KEY=xxx
OSS_AK=xxx
OSS_SK=xxx

```

## 三、核心模块设计

### 3.1 景点列表获取（腾讯地图）

使用腾讯地图 WebService API 的 **地点搜索** 接口：

- API: `https://apis.map.qq.com/ws/place/v1/search`
- 关键参数：
  - `keyword`: 搜索关键词
  - `boundary`: `region(松阳县, 0)` — 注意是 `boundary` 字段，不是 `region`
  - `page_size`: 20（最大值）
  - `page_index`: 从1开始
  - `key`: 腾讯地图Key

**搜索关键词列表**（扩充版）：
```python
KEYWORDS = [
    "景点", "旅游", "风景区", "古村", "古镇",
    "公园", "寺庙", "博物馆", "纪念馆",
    "古桥", "古道", "梯田", "廊桥",
    "红色景点", "革命遗址",
    "自然保护区", "湿地",
    "祠堂", "古建筑", "塔",
]
```

**分页终止条件**：
- 返回数量 < page_size
- page_index > 10（腾讯地图上限约200条/关键词）
- 累计景点数达到 max_spots 上限

**关键字段映射**：
- `title` → `attractionName`
- `location.lng` → `longitude`（GCJ-02坐标系）
- `location.lat` → `latitude`（GCJ-02坐标系）
- `id` → 用于去重

**坐标系说明**：腾讯地图返回的是GCJ-02坐标系（国测局加密坐标），直接存储。如果下游系统需要WGS-84坐标，需额外转换（本工具不做转换，在注释中说明）。

### 3.2 去重与过滤

**三层去重策略**：
1. 按腾讯地图POI `id` 去重（同一POI不同关键词命中）
2. 按名称精确匹配去重（同名不同id的情况）
3. 按坐标距离去重：两个景点经纬度距离 < 100米 且名称相似度 > 0.6，视为重复

**名称相似度算法**：使用 Jaro-Winkler 算法（对中文短文本表现较好），Python使用 `jellyfish` 库。示例：
- "石仓古民居群" vs "石仓古民居" → 相似度 ~0.95 → 去重
- "杨家堂村" vs "杨家堂古村落" → 相似度 ~0.78 → 去重
- "龙泉寺" vs "龙泉山" → 相似度 ~0.72 但坐标距离远 → 保留

**过滤规则**（排除非景点POI）：
```python
EXCLUDE_KEYWORDS = [
    "停车场", "游客中心", "售票处", "厕所", "酒店",
    "宾馆", "民宿", "餐厅", "饭店", "超市",
    "旅行社", "开发有限公司", "管理处",
]
```

利用腾讯地图返回的 `category` 字段做类目过滤，只保留旅游景点相关类目。

### 3.3 手动补充景点

支持从文件读取额外景点名称：
```
# extra_spots.txt
杨家堂村
呈回村
横樟村
酉田村
...
```

对于手动补充的景点，使用腾讯地图的 **地点搜索** 接口精确搜索获取经纬度：
- keyword: 景点名称
- boundary: `region(districtName, 0)`
- 取第一条结果的坐标

如果搜索不到，使用 **地址解析（geocoder）** 接口：
- API: `https://apis.map.qq.com/ws/geocoder/v1/`
- address: `provinceName + cityName + districtName + 景点名称`

如果地址解析也失败，**跳过该景点**并记录到日志，不使用近似坐标。

**与API数据合并规则**：
- 手动补充的景点如果与API搜索结果重名（经去重判定为同一景点），以API数据为准
- 手动补充的景点格式要求：每行一个景点名称，UTF-8编码，空行和#开头的行忽略

### 3.4 AI内容生成（Azure OpenAI）

**SDK配置**：

```python
from openai import OpenAI

client = OpenAI(
    base_url=os.getenv("AZURE_OPENAI_BASEURL"),
    api_key=os.getenv("AZURE_OPENAI_API_KEY"),
)
```

> 注：用户提供的 base_url 为 `https://jxy-us-2.cognitiveservices.azure.com/openai/v1`，
> 这是一个兼容 OpenAI API 格式的自定义代理端点，使用标准 `OpenAI` 客户端即可（非 AzureOpenAI 客户端）。
> 模型名称使用 `gpt-4.1` 或实际部署名称（运行时通过环境变量 `AZURE_OPENAI_MODEL` 配置，默认 `gpt-4.1`）。

### 3.5 图片生成（火山引擎 Seedream）

```python
import requests

headers = {
    "Content-Type": "application/json",
    "Authorization": f"Bearer {os.getenv('ARK_API_KEY')}"
}

payload = {
    "model": "doubao-seedream-5-0-260128",
    "prompt": f"中国{province_name}{city_name}{district_name}{attraction_name}，风景摄影，高清，自然光，旅游宣传照片风格，4K画质，真实感",
    "sequential_image_generation": "disabled",
    "response_format": "url",
    "size": "1024x1024",
    "stream": False,
    "watermark": True
}

response = requests.post(
    "https://ark.cn-beijing.volces.com/api/v3/images/generations",
    headers=headers,
    json=payload,
    timeout=120
)
```

**返回处理**：
- 如果 `response_format` 为 "url"，返回临时URL（有效期有限），**必须立即下载到本地**再上传OSS
- 如果返回 base64，需解码后上传
- 代码中同时处理两种情况

**图片下载流程**：
```python
# 获取图片URL后立即下载
image_url = result["data"][0].get("url")
if image_url:
    img_response = requests.get(image_url, timeout=30)
    image_data = img_response.content
else:
    # base64 fallback
    import base64
    image_data = base64.b64decode(result["data"][0]["b64_json"])
```

**图片下载安全**：
- 验证URL域名白名单（只允许火山引擎CDN域名）
- 检查Content-Type为image/*
- 文件大小上限10MB

### 3.6 OSS上传

```python
import oss2
import uuid
from datetime import datetime

auth = oss2.Auth(os.getenv('OSS_AK'), os.getenv('OSS_SK'))
bucket = oss2.Bucket(auth, f"https://{os.getenv('OSS_REGION')}.aliyuncs.com", os.getenv('OSS_BUCKET'))

now = datetime.now()
file_key = f"travel18/attraction/{now.strftime('%Y/%m/%d')}/{uuid.uuid4()}.jpg"

bucket.put_object(
    file_key,
    image_data,
    headers={'Content-Type': 'image/jpeg'}
)

url = f"https://{os.getenv('OSS_BUCKET')}.{os.getenv('OSS_REGION')}.aliyuncs.com/{file_key}"
```

## 四、执行流程

```
1. 加载环境变量（.env文件）
2. 解析命令行参数
3. 创建输出目录 ./{districtName}/
4. 加载断点进度（如有）
5. 调用腾讯地图API，多关键词搜索 → 原始景点列表
6. 加载手动补充景点（如有），逐个查询经纬度
7. 合并、去重、过滤 → 最终景点列表
8. [dry-run模式到此结束，输出景点列表]
9. 对每个未处理的景点：
   9.1 调用Azure OpenAI生成AI内容
   9.2 校验AI输出，修正异常值
   9.3 调用Seedream生成图片
   9.4 下载图片并上传OSS
   9.5 组装完整JSON
   9.6 追加写入 attractions.jsonl
   9.7 更新 progress.json
10. 输出统计信息（景点数、API调用次数、估算费用）
```

## 六、频率控制与重试

| 服务 | 限制 | 实现方式 |
|------|------|----------|
| 腾讯地图 | 5 QPS | time.sleep(0.3) 间隔（留余量） |
| Azure OpenAI | 串行调用 | 逐个处理，无并发 |
| Seedream | 1次/2秒 | time.sleep(2) 间隔 |
| OSS | 无特殊限制 | 正常调用 |

> 简化设计：不使用异步并发，全部串行处理。50个景点预计运行10-15分钟，可接受。
> 100个景点约30-40分钟（Seedream生成耗时较长）。

### 重试策略（指数退避）

所有API调用遇到 429/5xx 错误时，使用指数退避重试：

```python
MAX_RETRIES = 3
RETRY_DELAYS = [2, 4, 8]  # 秒

for attempt in range(MAX_RETRIES):
    try:
        response = call_api(...)
        break
    except (RateLimitError, ServerError) as e:
        if attempt < MAX_RETRIES - 1:
            time.sleep(RETRY_DELAYS[attempt])
        else:
            # 记录错误，跳过当前景点
            log_error(...)
```

**错误处理原则**：单个景点失败时跳过并记录到日志，不终止整体流程。

## 八、注意事项

1. **景点名称和经纬度禁止AI生成** — 必须来自腾讯地图API真实数据
2. **坐标系** — 存储的是GCJ-02坐标系（腾讯地图原生坐标），与高德地图一致，与WGS-84有偏移
3. **凭证安全** — 所有密钥通过环境变量注入，.env文件加入.gitignore
4. **JSONL格式** — 每行一个完整JSON对象，不是JSON数组
5. **字段类型严格** — longitude/latitude为float，评分类为string，门票为number或null
6. **AI内容声明** — 描述、评分等AI生成字段仅供参考，建议人工审核后使用
7. **覆盖率** — 工具无法保证获取"所有"景点，建议配合手动补充使用
8. **模型名称** — Azure OpenAI的模型名需要确认实际部署名称，默认使用 `gpt-4.1`，可通过环境变量 `AZURE_OPENAI_MODEL` 覆盖
